## Как начать тестирование

### 1. Откройте браузер
Перейдите на: `http://localhost:5173/`

### 2. Авторизуйтесь
**Страница входа**: http://localhost:5173/login

Учётные данные:
```
Email:    admin@test.local
Пароль:   admin123
```

### 3. Перейдите на страницу проектов
После авторизации вы попадёте на страницу `/projects`

Там должен быть проект `TEST` (ID: 2)

### 4. Откройте проект и создайте задачу

1. Нажмите на проект `TEST` 
2. Нажмите кнопку "Создать задачу"
3. Заполните форму:
   - **Статус**: выберите "To Do" (ID: 2)
   - **Название**: e.g. "Моя первая задача"
   - **Описание**: произвольный текст
   - **План старт/конец**: опционально (можно пропустить)
4. Нажмите "Создать"

### 5. Протестируйте CRUD операции

- **Создание**: откройте проект → "Создать задачу"
- **Редактирование**: на странице задачи → "Редактировать"
- **Удаление**: на странице задачи → "Удалить"

## Если что-то не работает

### 1. Проверьте логи бэкенда
```bash
docker compose -f infra/docker-compose.yml logs backend --tail=100
```

### 2. Проверьте логи фронтенда
```bash
docker compose -f infra/docker-compose.yml logs frontend --tail=100
```

### 3. Перезагрузите стек
```bash
docker compose -f infra/docker-compose.yml down
docker compose -f infra/docker-compose.yml up -d
```

### 4. Очистите локальное хранилище браузера
Откройте DevTools (F12) → Application → Local Storage → Удалите `ast3_access` и `ast3_refresh`

## Возможные проблемы и решения

### Проблема: "Ошибка авторизации"
**Решение**: Убедитесь, что вводите корректно:
- Email: `admin@test.local` (не username)
- Пароль: `admin123`

### Проблема: При создании задачи выбрать статус не удаётся
**Решение**: Убедитесь, что в поле "Project ID" вводится ID проекта: `2`

### Проблема: "Не удалось загрузить статусы проекта"
**Решение**: Проверьте, что project_id введён правильно и проект существует

### Проблема: CORS ошибка
**Решение**: Убедитесь, что CORS настроен в docker-compose.yml для localhost:5173

## Команды Docker для быстрого доступа

```bash
# Просмотр логов
docker compose -f infra/docker-compose.yml logs -f

# Перезапуск всего стека
docker compose -f infra/docker-compose.yml restart

# Вход в контейнер бэкенда
docker compose -f infra/docker-compose.yml exec backend bash

# Запуск произвольной команды в бэкенде
docker compose -f infra/docker-compose.yml exec backend python manage.py shell

# Просмотр статуса контейнеров
docker compose -f infra/docker-compose.yml ps
```
